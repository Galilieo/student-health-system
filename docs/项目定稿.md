# 乐健项目定稿

## 项目名称

乐健（面向大学生的智能健康习惯管理系统）

## 项目概述

“乐健”面向大学生群体，围绕饮水、睡眠、运动等日常健康行为，提供记录、反馈与习惯养成支持。系统通过打卡记录、历史查询与统计分析，帮助用户形成规律生活方式。

## 项目目标

- 完成学生端健康习惯管理的核心闭环：登录/注册 -> 打卡 -> 历史 -> 统计
- 提供基础提醒与建议能力，增强持续使用效果
- 建立管理端最小可用模块，支持用户、公告、反馈管理

## 核心功能

### 学生端功能

- 登录与注册
- 首页健康概览
- 每日打卡（饮水、睡眠、运动）
- 历史记录查看
- 统计分析
- 个人中心（目标设置、基础资料）
- 健康提醒
- 智能建议（可先规则版）

### 管理端功能

- 用户信息管理
- 公告管理
- 用户反馈管理

## 主要页面

- 登录页
- 注册页
- 首页
- 每日打卡页
- 历史记录页
- 统计分析页
- 个人中心页
- 提醒设置页
- 智能建议页
- 管理员后台页（用户/公告/反馈）

## 技术栈

- Android：Java + XML
- 后端：Spring Boot
- 数据库：MySQL
- AI能力：健康建议生成（扩展能力）

## 开发原则

- 所有人在 `dev` 分支开发，`main` 作为稳定版本
- 前后端统一接口返回结构：`{ code, message, data }`
- 按模块分工，减少多人同时修改同一文件
- 每周进行阶段复盘，及时处理风险与阻塞

## 阶段安排（第7周-第13周）

- 第7-8周：前端主流程闭环，认证模块（登录+注册）完善，数据库核心表结构完成
- 第9-10周：核心接口联调（登录、注册、打卡、概览、历史）
- 第11周：功能完善（统计、提醒、个人中心）与UI统一优化
- 第12周：测试与缺陷修复，管理端最小功能（用户/公告/反馈）验收
- 第13周：最终验收与项目交付

## 里程碑

- Milestone 1（第8周末）：完成认证闭环与前端主流程本地版
- Milestone 2（第10周末）：完成核心接口联调并切换真实数据
- Milestone 3（第11周末）：完成统计分析、提醒设置、个人中心
- Milestone 4（第12周末）：完成管理端最小功能并通过回归测试
- Milestone 5（第13周末）：完成最终版本验收与项目交付

## 验收标准

- 主流程可完整演示：登录/注册 -> 首页 -> 打卡 -> 历史 -> 退出登录
- 核心接口可稳定使用：`/login`、`/register`、`/checkin`、`/summary/today`、`/history`
- 主要页面可用，功能逻辑闭环
- 文档齐全，版本可交付

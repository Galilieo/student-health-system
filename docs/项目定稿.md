# 乐健项目定稿

## 1. 项目概述

“乐健”是一款面向大学生群体的智能健康习惯管理系统，围绕“记录-反馈-养成”构建完整使用闭环。系统以日常打卡为核心，支持饮水、睡眠、运动等健康行为记录，并通过历史记录与统计分析帮助用户形成规律生活方式。

## 2. 技术栈

- Android：Java + XML
- 后端：Spring Boot
- 数据库：MySQL
- AI能力：健康建议生成（扩展模块）

## 4. 详细开发计划（第7周-第13周）

### 4.1 第一阶段：功能闭环完善（第7-8周）

- 安卓端完成登录、首页、打卡、历史的本地版闭环
- 明确后端接口字段与返回结构（code/message/data）
- 统一认证方式（Bearer Token）

### 4.2 第二阶段：前后端联调（第9-10周）

- 联调接口：`POST /login`、`POST /checkin`、`GET /summary/today`、`GET /history`
- 完成异常处理：网络异常、业务失败、token失效
- 将前端本地数据源切换为真实接口数据

### 4.3 第三阶段：功能完善与优化（第11周）

- 完成统计分析页基础展示
- 完成个人中心基础信息展示
- 统一页面视觉规范与交互反馈

### 4.4 第四阶段：测试与缺陷修复（第12周）

- 功能测试：登录、打卡、历史、退出登录
- 接口测试：参数校验、边界值、异常码
- 兼容性测试：不同屏幕尺寸与Android版本
- 修复高优先级缺陷并完成回归

### 4.5 第五阶段：验收与交付（第13周）

- 完成项目说明文档、测试报告、部署说明
- 完成版本验收与最终提交

## 5. 里程碑规划

- Milestone 1（第8周）：前端本地版主流程完成，接口文档冻结
- Milestone 2（第10周）：前后端核心接口联调完成
- Milestone 3（第11周）：功能完善与UI统一完成
- Milestone 4（第12周）：测试完成并关闭关键缺陷
- Milestone 5（第13周）：最终验收与版本交付

## 6. 风险与应对

- 后端接口延期：前端保留mock与接口预留方法，接口到位即切换
- 联调字段不一致：提前冻结字段与错误码，统一接口示例
- 时间压力大：优先保障“登录-打卡-历史-展示”主链路

## 7. 验收标准

- 可完整演示主流程：登录 -> 首页 -> 打卡 -> 历史 -> 退出登录
- 核心接口稳定可用（登录、打卡、概览、历史）
- 测试记录与项目文档完整
- 第13周前完成可交付版本
